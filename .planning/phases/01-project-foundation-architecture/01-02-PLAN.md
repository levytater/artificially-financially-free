**1. Types (`src/types/calculator.ts`):**
Define `CalculatorState` interface with all initial calculator inputs:
- `purchasePrice: number` (default 600000)
- `downPaymentPercent: number` (default 20)
- `mortgageRate: number` (default 5.5)
- `amortizationYears: number` (default 25)
- `monthlyRent: number` (default 2000)
- `province: string` (default 'ON')
- `timeHorizon: number` (default 25)
- `firstTimeBuyer: boolean` (default false)
- `annualIncome: number` (default 75000)

Define `CalculatorContextValue` interface:
- `state: CalculatorState`
- `setState: (updates: Partial<CalculatorState>) => void`
- `resetToDefaults: () => void`

Add a comment that more inputs will be added in Phase 4 (master return dial, per-account returns, etc.).

**2. Defaults (`src/lib/defaults.ts`):**
Export `calculatorDefaults: CalculatorState` object with the default values listed above. These are sensible Canadian defaults.

**3. Parsers (`src/lib/parsers.ts`):**
Using nuqs, define `calculatorParsers` object with type-safe parsers:
- `parseAsInteger` for price, rent, amort, timeHorizon, income
- `parseAsFloat` for downPaymentPercent, mortgageRate
- `parseAsString` for province
- `parseAsBoolean` for firstTimeBuyer
- Each parser `.withDefault(value)` matching the defaults

Define `calculatorUrlKeys` mapping verbose keys to short URL params:
- purchasePrice -> 'price', downPaymentPercent -> 'dp', mortgageRate -> 'rate', amortizationYears -> 'amort', monthlyRent -> 'rent', province -> 'province', timeHorizon -> 'years', firstTimeBuyer -> 'ftb', annualIncome -> 'income'

Import types from `nuqs/server` (not `nuqs`) for server-safe parser definitions.

**4. Serializer (`src/lib/serializer.ts`):**
Create `serializeCalculatorState` using `createSerializer` from `nuqs/server` with `calculatorParsers` and `{ clearOnDefault: true, urlKeys: calculatorUrlKeys }`. Only non-default values will appear in generated URLs.

**5. Decimal.js (`src/lib/decimal.ts`):**
Configure Decimal.js:
- `precision: 20`
- `rounding: Decimal.ROUND_HALF_UP`
- `toExpNeg: -9`
- `toExpPos: 21`

Export helpers:
- `effectiveMonthlyRate(annualRate: number): Decimal` -- Canadian semi-annual compounding: `(1 + annual/2)^(1/6) - 1`
- `formatCurrency(value: Decimal | number): string` -- `.toFixed(2)`

**6. Calculator State Hook (`src/hooks/use-calculator-state.ts`):**
'use client' hook using `use-local-storage-state`:
- Store key: `'aff-calculator-state'`
- Default value: `calculatorDefaults`
- Returns `{ state, setState (merge partial), resetToDefaults }`
- `setState` merges partial updates into existing state (spread pattern)
- Uses `useLocalStorageState` from `use-local-storage-state` package

**7. Calculator Provider (`src/providers/calculator-provider.tsx`):**
'use client' React Context provider:
- Creates `CalculatorContext` with `createContext<CalculatorContextValue | null>(null)`
- `CalculatorProvider` component wraps `useCalculatorState` hook and provides context value
- Exports `useCalculator()` hook that reads from context (throws if used outside provider)
- This is the single source of truth -- any component (or future AI chatbot) calls `useCalculator()` to read/write state

**8. Share URL Hook (`src/hooks/use-share-url.ts`):**
'use client' hook:
- Imports `serializeCalculatorState` from serializer
- Imports `useCalculator` from provider
- `generateShareUrl()` -- returns full URL string using `window.location.origin`
- `copyShareUrl()` -- copies to clipboard via `navigator.clipboard.writeText`

**9. Content Data Files (ARCH-03):**

`src/content/tooltips.ts`:
- Export `TooltipContent` interface: `{ label: string, description: string, learnMore?: string }`
- Export `tooltips: Record<string, TooltipContent>` with entries for ALL initial inputs:
  - purchasePrice, downPaymentPercent, mortgageRate, amortizationYears, monthlyRent, province, timeHorizon, firstTimeBuyer, annualIncome
- Descriptions use markdown for bold/links (e.g., "**minimum is 5%**", "[CMHC insurance](#cmhc)")
- Content should be accurate Canadian financial information. See RESEARCH.md for example content.

`src/content/explainers.ts`:
- Export `ExplainerContent` interface: `{ title: string, slug: string, body: string, relatedInputs: string[] }`
- Export `explainers: Record<string, ExplainerContent>` with initial entries:
  - cmhc (CMHC mortgage insurance)
  - semi-annual-compounding (Canadian mortgage compounding)
  - opportunity-cost (renting + investing the difference)
  - land-transfer-tax (provincial LTT overview)
  - marginal-tax-rates (how investment gains are taxed)
- Body fields use markdown. relatedInputs reference tooltip keys.

`src/content/verdict-text.ts`:
- Export `VerdictTextContent` interface: `{ scenario: string, headline: string, summary: string, factors: string[] }`
- Export `verdictText: Record<string, VerdictTextContent>` with two entries:
  - 'rent-wins': headline, summary template, and key factors when renting is better
  - 'buy-wins': headline, summary template, and key factors when buying is better
- Templates use `{{placeholder}}` syntax for dynamic values (filled in Phase 5)

Add `CalculatorProvider` to the provider hierarchy. Final nesting: ThemeProvider -> NuqsAdapter -> CalculatorProvider -> {children}
