# Phase 05.1 Plan 01: Result Tooltips & Visual Hierarchy Summary

**One-liner:** InfoTooltip component and result metric explanations with enhanced dollar advantage emphasis and income-based tax rate context

## What Was Built

All result display components now include inline help tooltips explaining financial metrics in plain language. The dollar advantage card is visually emphasized as the primary decision metric. The verdict card explains where the marginal tax rate comes from using the user's actual income and province. The theme toggle now shows what mode you'll switch TO rather than icon-only.

### Components Created

**InfoTooltip Component**
- Reusable wrapper around shadcn/ui Tooltip primitives
- Info icon (lucide-react) as trigger button
- Accessible with keyboard navigation and screen reader support
- Collision detection with viewport boundaries for mobile
- Used consistently across all result metrics

### Components Enhanced

**Summary Cards (2x2 Grid)**
- InfoTooltip added to all 4 card labels
- Dollar advantage card: `border-2` (was `border`) and `text-3xl` (was `text-2xl`)
- Tooltips explain: renter net worth, buyer net worth, dollar advantage, percentage advantage
- Visual hierarchy now clearly emphasizes the bottom-line number

**Verdict Card**
- Tax rate explanation section added below verdict text
- Shows: "Based on your $X income in [Province], your rate is Y%"
- InfoTooltip on break-even year text
- InfoTooltip on marginal tax rate explanation
- Users understand calculation provenance

**Theme Toggle**
- Changed from `size="icon"` to `size="sm"` to accommodate text
- Text label shows "Light" when in dark mode (what you'll get), "Dark" in light mode
- Icon animation preserved
- Reduces confusion about toggle behavior

**Tooltip Content (tooltips.ts)**
- 6 new result metric entries added:
  - `renterNetWorth`: Portfolio composition explanation
  - `buyerNetWorth`: Net proceeds after selling calculation
  - `dollarAdvantage`: Absolute wealth difference definition
  - `percentageAdvantage`: Contextualized significance metric
  - `breakEvenYear`: When buying surpasses renting (or vice versa)
  - `marginalTaxRate`: Combined federal + provincial rate explanation

**App Layout**
- `TooltipProvider` added at root level inside `CalculatorProvider`
- `delayDuration={200}` for responsive tooltip display
- Prevents hydration mismatch by using single provider for entire app
