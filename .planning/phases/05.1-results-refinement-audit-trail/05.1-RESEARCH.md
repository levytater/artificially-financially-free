## Summary

Phase 5.1 addresses 8 gap-closure improvements identified during Phase 5 visual verification checkpoint: dollar advantage styling, income explanation, comprehensive helper tooltips, detailed audit/math breakdown tab, print/PDF support, theme toggle UX, enhanced visual hierarchy, and explanatory content. The phase focuses on transparency, accessibility, and professional presentation of financial calculation results.

This research covers three main technical domains: (1) accessible tooltip implementation using existing shadcn/ui Radix primitives, (2) print/PDF export using browser-native CSS media queries with react-to-print for enhanced control, and (3) audit trail UI patterns from fintech best practices. All solutions integrate with the existing Next.js + TypeScript + Tailwind CSS + shadcn/ui stack without introducing new dependencies beyond react-to-print.

**Primary recommendation:** Use shadcn/ui Tooltip (already installed) for helper tooltips, implement browser-native print with CSS @media print queries enhanced by react-to-print v3.3.0 for PDF export, and create a dedicated "Math Breakdown" tab with year-by-year audit tables matching the Excel reference structure.

### Pattern 1: Reusable Info Tooltip Wrapper

**What:** Generic `<InfoTooltip>` component wrapping shadcn/ui Tooltip primitives with info icon trigger
**When to use:** For inline help tooltips next to labels, result values, and table headers
**Example:**

```typescript
// src/components/ui/info-tooltip.tsx
import { Info } from 'lucide-react'
import { Tooltip, TooltipContent, TooltipProvider, TooltipTrigger } from '@/components/ui/tooltip'

interface InfoTooltipProps {
  content: string
  side?: 'top' | 'bottom' | 'left' | 'right'
  className?: string
}

export function InfoTooltip({ content, side = 'top', className }: InfoTooltipProps) {
  return (
    <TooltipProvider delayDuration={200}>
      <Tooltip>
        <TooltipTrigger asChild>
          <button
            type="button"
            className={cn(
              "inline-flex h-4 w-4 items-center justify-center rounded-full text-muted-foreground hover:text-foreground transition-colors",
              className
            )}
            aria-label="More information"
          >
            <Info className="h-3.5 w-3.5" />
          </button>
        </TooltipTrigger>
        <TooltipContent side={side} className="max-w-xs">
          <p className="text-xs leading-relaxed">{content}</p>
        </TooltipContent>
      </Tooltip>
    </TooltipProvider>
  )
}
```

**Usage in result components:**
```tsx
<div className="flex items-center gap-2">
  <span className="text-sm font-medium">Marginal Tax Rate</span>
  <InfoTooltip content={tooltips.marginalTaxRate.description} />
  <span className="text-lg font-bold">{formatPercentageDecimal(result.marginalTaxRate)}</span>
</div>
```

### Pattern 3: Print Button with State Management

**What:** useReactToPrint hook with onBeforePrint/onAfterPrint callbacks for async state updates
**When to use:** For "Export PDF" button that customizes print window before opening dialog
**Example:**

```typescript
// src/components/results/export-pdf-button.tsx
'use client'

import { useState, useRef, useEffect } from 'react'
import { useReactToPrint } from 'react-to-print'
import { Printer } from 'lucide-react'
import { Button } from '@/components/ui/button'

export function ExportPdfButton() {
  const [isPrinting, setIsPrinting] = useState(false)
  const contentRef = useRef<HTMLDivElement>(null)
  const promiseResolveRef = useRef<(() => void) | null>(null)

  // Wait for isPrinting state to update before resolving print promise
  useEffect(() => {
    if (isPrinting && promiseResolveRef.current) {
      promiseResolveRef.current()
    }
  }, [isPrinting])

  const handlePrint = useReactToPrint({
    contentRef,
    documentTitle: `Rent-vs-Buy-Analysis-${new Date().toISOString().split('T')[0]}`,
    onBeforePrint: () => {
      // Return Promise to wait for state update
      return new Promise<void>((resolve) => {
        promiseResolveRef.current = resolve
        setIsPrinting(true)
      })
    },
    onAfterPrint: () => {
      promiseResolveRef.current = null
      setIsPrinting(false)
    }
  })

  return (
    <>
      <Button onClick={handlePrint} variant="outline" className="no-print">
        <Printer className="mr-2 h-4 w-4" />
        Export PDF
      </Button>
      <div ref={contentRef} className="print-full-width">
        {/* All printable content */}
      </div>
    </>
  )
}
```

**Source:** Context7 - react-to-print Promise-based state management

### Anti-Patterns to Avoid

- **Server-side PDF generation for v1:** Adds backend complexity (Modal/serverless functions) when browser print-to-PDF is sufficient. Defer to v2 if premium PDF templates needed.
- **Tooltip on every word:** Over-tooltipping creates fatigue. Use tooltips only for financial jargon, calculation assumptions, and non-obvious inputs.
- **Breaking print layout with fixed positioning:** Fixed headers/footers don't paginate correctly. Use static layout for print styles.
- **Hardcoding tooltip content in JSX:** Violates ARCH-03 requirement. Store all explanatory content as TypeScript objects in src/content/ for future AI chatbot consumption.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Accessible tooltips from scratch | Custom tooltip with manual ARIA, positioning, keyboard nav | shadcn/ui Tooltip (Radix primitives) | Already installed, WAI-ARIA compliant, handles edge detection, focus management, screen reader announcements |
| PDF generation library | Custom PDF renderer with canvas/SVG | Browser print with CSS @media print | Native browser support, no bundle size increase, users familiar with print dialog, "Save as PDF" built-in |
| Print window customization | Manual window.print() + DOM manipulation | react-to-print with callbacks | Handles async state updates, provides lifecycle hooks, manages focus restoration |
| Complex positioning logic for tooltips | Reinvent Floating UI calculations | Radix Tooltip with built-in positioning | Handles collision detection, viewport boundaries, portal rendering automatically |

**Key insight:** Browser-native print combined with CSS media queries is the most maintainable solution for PDF export. React-to-print adds conveniences (callbacks, state management) without requiring a complete rewrite. Accessible tooltips are hard to get right—Radix primitives solve edge cases (keyboard nav, screen readers, touch devices) that custom implementations miss.

## Common Pitfalls

### Pitfall 1: Tooltip Positioning Overflow on Mobile

**What goes wrong:** Tooltips render off-screen on small viewports when positioned "top" or "right"
**Why it happens:** Fixed positioning doesn't account for viewport boundaries without collision detection
**How to avoid:** Use Radix Tooltip's built-in collision detection with `collisionPadding` prop
**Warning signs:** Tooltips cut off at edge of screen, horizontal scrollbars appear on mobile

**Prevention:**
```typescript
<TooltipContent side="top" collisionPadding={8}>
  {/* Content automatically repositions if hitting viewport edge */}
</TooltipContent>
```

### Pitfall 2: Print Styles Not Applied in Production Build

**What goes wrong:** Print styles work in dev but fail in production due to CSS purging
**Why it happens:** Tailwind CSS purges unused classes; print: variant may be stripped if not safeguarded
**How to avoid:** Explicitly list print classes in Tailwind config or use @media print in globals.css
**Warning signs:** Print preview shows unstyled content, elements not hidden/shown as expected

**Prevention:**
```javascript
// tailwind.config.js
module.exports = {
  content: ['./src/**/*.{ts,tsx}'],
  safelist: [
    'print:hidden',
    'print:block',
    'print:table-cell',
  ],
}
```

Or use @media print in globals.css (safer, not subject to purging).

### Pitfall 3: Theme Toggle Icon Confusion (Current State vs. Future State)

**What goes wrong:** Users confused whether icon shows "current theme" or "what happens on click"
**Why it happens:** No consistent industry standard—some apps show current state, others show action
**How to avoid:** Show the icon of the mode you'll switch TO (sun in dark mode = "click to go light")
**Warning signs:** User testing reveals confusion, users toggle multiple times expecting different result

**Current implementation in theme-toggle.tsx:**
- Shows Moon in light mode (click to go dark) ✅
- Shows Sun in dark mode (click to go light) ✅
- Correct pattern per research (action-oriented)

**Enhancement:** Add descriptive aria-label/title attribute:
```typescript
<Button
  onClick={toggleTheme}
  aria-label={`Switch to ${theme === 'dark' ? 'light' : 'dark'} mode`}
  title={`Switch to ${theme === 'dark' ? 'light' : 'dark'} mode`}
>
```

### Pitfall 4: Hydration Mismatch with TooltipProvider

**What goes wrong:** React hydration error when TooltipProvider wraps server-rendered content
**Why it happens:** Tooltip state initialized on client doesn't match server HTML
**How to avoid:** Wrap app root in TooltipProvider once in layout.tsx, not per-component
**Warning signs:** Console error "Text content does not match server-rendered HTML", tooltips flash on load

**Prevention:**
```typescript
// src/app/layout.tsx
import { TooltipProvider } from '@/components/ui/tooltip'

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <ThemeProvider>
          <TooltipProvider delayDuration={200}>
            {children}
          </TooltipProvider>
        </ThemeProvider>
      </body>
    </html>
  )
}
```

### Pitfall 5: Audit Table Overload (Too Many Columns)

**What goes wrong:** Year-by-year audit table with 20+ columns becomes unusable on any screen size
**Why it happens:** Trying to show every intermediate calculation in a single table
**How to avoid:** Group related columns under expandable sections, or create multiple focused tables
**Warning signs:** Horizontal scroll exceeds 2x viewport width, users can't correlate rows

**Prevention:** Break into 3 focused tables:
1. **Home Equity Growth** (5 columns): Year, Home Value, Mortgage Balance, Equity, Annual Appreciation
2. **Cost Comparison** (5 columns): Year, Buyer Annual Cost, Renter Annual Rent, Monthly Savings, Tax Paid
3. **Net Worth Progression** (4 columns): Year, Renter Portfolio, Buyer Equity (w/ selling), Difference

Or use Accordion with expandable detail rows.

### Pitfall 6: Income Tax Explanation Missing

**What goes wrong:** Users see "Marginal Tax Rate: 43.7%" with no explanation of how it's calculated
**Why it happens:** Phase 3 auto-calculates combined federal + provincial tax from income, but UI doesn't explain
**How to avoid:** Add InfoTooltip next to tax rate showing: "Based on your $X income in [Province], combined federal (Y%) + provincial (Z%) = 43.7%"
**Warning signs:** User confusion in testing, questions like "Where does this tax rate come from?"

**Prevention:**
```tsx
<div className="flex items-center gap-2">
  <span className="text-sm text-muted-foreground">Marginal Tax Rate</span>
  <InfoTooltip
    content={`Based on your ${formatCurrencyDecimal(annualIncome)} annual income in ${province}, your combined federal (${formatPercentage(federalRate)}) + provincial (${formatPercentage(provincialRate)}) marginal tax rate is ${formatPercentage(marginalTaxRate)}. This affects taxation on your investment gains as a renter.`}
  />
  <span className="text-base font-medium">{formatPercentageDecimal(marginalTaxRate)}</span>
</div>
```

## State of the Art

| Old Approach | Current Approach | When Changed | Impact |
|--------------|------------------|--------------|--------|
| Global tooltip libraries (Tippy.js standalone) | Radix UI primitives via shadcn/ui | 2023-2024 | Better accessibility, smaller bundle (primitives not full library), consistent with shadcn ecosystem |
| Server-side PDF generation for all use cases | Browser print with CSS media queries | 2020+ | Client-side reduces infrastructure needs, faster for simple reports, users control print settings |
| Manual keyboard navigation in tooltips | WAI-ARIA compliant primitives | 2021+ | Screen reader support built-in, focus management automatic, reduced accessibility bugs |
| Page-break-* CSS properties | break-after, break-before, break-inside | 2023+ | Better browser support, more intuitive API, less deprecated warnings |

**Deprecated/outdated:**
- `react-to-print` v2 API: v3 (released 2024) uses `contentRef` instead of `content` callback, requires migration if older tutorials referenced
- Tippy.js for React tooltips: Radix UI is now the industry standard for headless accessible components
- Print-specific link elements: Modern @media print in main stylesheet is cleaner than separate print.css

## Open Questions

1. **Excel audit structure specifics**
   - What we know: User has reference Excel file "Shareable - 2025 Rent vs Buy Investment Calculator.xlsx" in project root
   - What's unclear: Exact column headers, groupings, and row structure (file is binary .xlsx, not readable via Read tool)
   - Recommendation: During planning, manually inspect Excel file to map exact audit table structure. Key columns likely: Year, Home Value, Mortgage Balance, Home Equity, Annual Buyer Costs, Annual Rent, Monthly Savings, Portfolio Balance, Tax Paid, Net Worth Difference.

2. **Tooltip content authoring process**
   - What we know: All content must be stored as TypeScript objects per ARCH-03, existing tooltips in src/content/tooltips.ts
   - What's unclear: Whether to write all tooltip content upfront or incrementally as components are built
   - Recommendation: Create tooltip content stubs during Wave 0 (content authoring) with placeholders, then refine during component implementation based on actual UI context.

3. **Print page breaks for multi-year audit tables**
   - What we know: CSS `page-break-inside: avoid` prevents awkward table splits
   - What's unclear: How many years fit on one page (depends on paper size, margins, font size)
   - Recommendation: Test with 10-year, 20-year, and 30-year scenarios. If >15 years causes multi-page table, consider pagination or accordion grouping (e.g., Years 1-10, 11-20, 21-30).
