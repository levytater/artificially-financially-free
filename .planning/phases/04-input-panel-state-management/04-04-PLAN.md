**Create `src/hooks/use-debounced-comparison.ts`:**
- `'use client'` directive
- Import `useEffect`, `useState` from React
- Import `useCalculator` from `@/providers/calculator-provider`
- Import `calculateRentVsBuyComparison` from `@/lib/calculations/comparison`

- Import `ComparisonInput` type from `@/types/investment`

Hook signature: `useDebouncedComparison(): { results: ComparisonResult | null; isCalculating: boolean }`

Implementation:
- Read `state` from `useCalculator()`
- Maintain local `results` state (ComparisonResult | null, initially null) and `isCalculating` state (boolean)
- `useEffect` watching `state`:
  1. Set `isCalculating = true`
  2. Start a `setTimeout` with 300ms delay (per RESEARCH.md recommendation)
  3. Inside timeout callback:
     - Build `ComparisonInput` from the calculator state:
       ```typescript
       const input: ComparisonInput = {
         purchasePrice: state.purchasePrice,
         downPaymentPercent: state.downPaymentPercent,
         mortgageRate: state.mortgageRate,
         amortizationYears: state.amortizationYears,
         province: state.province as ProvinceCode,
         timeHorizon: state.timeHorizon,
         firstTimeBuyer: state.firstTimeBuyer,
         monthlyRent: state.monthlyRent,
         rentIncreasePercent: state.rentIncreaseRate,
         annualIncome: state.annualIncome,
         investmentReturnPercent: state.advancedMode
           ? state.investmentReturn  // In advanced mode, master dial is used for now; per-account TBD in future
           : state.investmentReturn,
         inflationRatePercent: state.inflationRate,
         // Optional overrides from advanced mode
         propertyTaxRate: undefined,  // Uses province default
         maintenanceRate: state.advancedMode ? state.maintenancePercent / 100 : undefined,
         homeInsurance: state.advancedMode ? state.homeInsurance : undefined,
         appreciationRate: state.advancedMode ? state.appreciationRate / 100 : undefined,
       }
       ```
       **Important note on rate conversion:** The ComparisonInput's optional fields (`maintenanceRate`, `appreciationRate`) expect decimal rates (0.01 for 1%), NOT percentage rates. The CalculatorState stores percentages (1.5 for 1.5%). Convert by dividing by 100 when passing to ComparisonInput. Check the housing-projection.ts to verify which format it expects -- look at the existing `DEFAULT_MAINTENANCE_RATE = 0.01` pattern. If ComparisonInput passes through to housing-projection and those optional fields expect percentage format (like other inputs), then do NOT divide. **Read the ComparisonInput -> HousingProjectionInput -> calculation function chain to determine the correct format.**
     - Wrap `calculateRentVsBuyComparison(input)` in a try/catch (validation errors from calculation functions should not crash the UI)
     - On success: set results state
     - On error: set results to null (or keep previous results) and log error to console
     - Set `isCalculating = false`
  4. Cleanup function: `clearTimeout(timer)` to cancel pending calculation if state changes again before 300ms

- Return `{ results, isCalculating }`
