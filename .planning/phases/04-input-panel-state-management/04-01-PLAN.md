**Create `src/lib/validation.ts`** with these validation functions:

```typescript
import type { CalculatorState } from '@/types/calculator'

export interface ValidationError {
  field: keyof CalculatorState
  message: string
}
```

Individual field validators (each returns `string | undefined` -- undefined means valid):

- `validatePurchasePrice(price: number)`: Must be > 0 and <= 10,000,000. Error messages: "Purchase price must be greater than $0", "Purchase price cannot exceed $10 million"
- `validateDownPaymentPercent(percent: number, purchasePrice: number)`: Canadian minimum down payment rules:
  - Homes <= $500K: minimum 5%
  - Homes $500K-$1M: 5% on first $500K + 10% on remainder (calculate as percentage of total)
  - Homes > $1M: minimum 20%
  - Maximum: 100%
  - Error messages include the calculated minimum percentage
- `validateMortgageRate(rate: number)`: 0-20% range. Cannot be negative.
- `validateAmortization(years: number)`: 1-30 years, integer only
- `validateTimeHorizon(years: number)`: 1-30 years
- `validateMonthlyRent(rent: number)`: >= 0, <= 50,000
- `validateAnnualIncome(income: number)`: >= 0, <= 10,000,000
- `validatePercentageRate(rate: number, fieldName: string)`: Generic percentage validator for appreciation, rent increase, inflation, maintenance, investment returns. Must be >= 0 and <= 50%. Error: "{fieldName} must be between 0% and 50%"
- `validateHomeInsurance(amount: number)`: >= 0, <= 50,000
- `validateSellingCosts(percent: number)`: 0-20% range

Aggregate validator:
- `validateCalculatorState(state: CalculatorState)`: Runs all individual validators, returns `ValidationError[]` (empty = valid)

Helper to get error for a single field:
- `validateField(field: keyof CalculatorState, value: number, state: CalculatorState)`: Returns `string | undefined` by dispatching to the appropriate validator based on field name
