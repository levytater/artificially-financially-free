**Create `src/components/input-panel/stepper-input.tsx`** -- the generic base component that CurrencyInput and PercentageInput will wrap:

Props interface `StepperInputProps`:
- `value: number` -- the current numeric value (from calculator state)
- `onChange: (value: number) => void` -- callback to update state
- `step: number` -- increment/decrement amount per arrow click
- `min?: number` -- minimum allowed value
- `max?: number` -- maximum allowed value
- `formatDisplay?: (value: number) => string` -- format for display (default: `v.toString()`)
- `parseInput?: (input: string) => number` -- parse user input to number (default: strip non-numeric, parseFloat)
- `label: string` -- accessible label text
- `error?: string` -- inline error message (red text below input)
- `suffix?: string` -- optional text suffix displayed after the input (e.g., "years", "%")
- `className?: string` -- additional container classes
- `id?: string` -- HTML id for the input element

Implementation details:
- Use `'use client'` directive at top
- Import `Input` from `@/components/ui/input` and `Button` from `@/components/ui/button`
- Import `ChevronUp` and `ChevronDown` from `lucide-react`
- Maintain local `displayValue` (string) state and `isFocused` (boolean) state
- **On focus:** Show raw number value (strip formatting) for easy editing
- **On blur:** Parse the local displayValue, call onChange with the parsed number, update display with formatted value. If parse fails (NaN), revert to formatted current value
- **On change (keyboard):** Only update local displayValue string, NOT the global state (prevent lag)
- **useEffect:** When `value` changes externally AND input is not focused, update displayValue to formatted version
- **Stepper buttons:** Positioned as a vertical stack (ChevronUp / ChevronDown) to the right of the input field. Each 20x20px (`h-5 w-5`), `variant="ghost"` for subtle appearance. Disable up when at max, disable down when at min. Use `tabIndex={-1}` to exclude from tab order (keyboard users use arrow keys or direct typing).
- **Clamp:** After increment/decrement, clamp to min/max if provided
- **Error display:** Red text below the input with `role="alert"` and `aria-describedby` linking to error element
- **Accessibility:** Input has `aria-invalid={!!error}`, `aria-label={label}`, and `aria-describedby` pointing to error element ID

The layout should be: label on top, then a row with [input field | stepper buttons column], then error message below. If suffix is provided, show it between the input and stepper buttons.

Apply dark theme styling: inputs use the existing shadcn `Input` component styles (which inherit from the Phantom dark theme). Stepper buttons should have subtle borders matching the theme.
